💡 Hyperparameter Tuning 101: Rank, Alpha & Learning Rate

Understanding the key hyperparameters for fine-tuning LLMs with QLoRA:

1. LoRA Rank ($r$)

Determines the inner dimension of the adapter matrices $A \in \mathbb{R}^{r \times d_{in}}$ and $B \in \mathbb{R}^{d_{out} \times r}$.

  • Rank 8–16: Ideal for style adaptations, domain vocabulary, or general instruction-following. Minimal VRAM overhead.
  • Rank 32–64: Best for complex reasoning, code generation, or heavy domain adaptation. Uses moderately more VRAM.

2. LoRA Alpha ($\alpha$)

Scaling factor applied to adapter gradient updates ($\Delta W = \frac{\alpha}{r} B \cdot A$).

  • Standard rule of thumb: set $\alpha = 2 \times r$ (e.g. Rank 16 $\rightarrow$ Alpha 32).

3. Learning Rate ($lr$)

Step size for parameter updates. Standard QLoRA recommendation is 0.0001 to 0.0003 (1e-4 to 3e-4).
